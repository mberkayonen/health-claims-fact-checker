# Established Science тАФ Further Reading Sources Design Spec

**Date:** 2026-05-28
**Status:** Approved

## Problem

`generateEstablishedVerdict` returns `sources: []`. The explanation comes entirely from Claude's parametric knowledge, leaving the user with nothing to explore further. Everyday users who want to dig deeper have no starting point.

## Goal

Attach real peer-reviewed sources to Established Science verdicts, presented honestly as "further reading" тАФ not as the basis of the verdict (which remains Claude's plain-language confirmation of foundational biology).

---

## Approach

### API route тАФ parallel search (changed)

In the established branch, replace the single `await generateEstablishedVerdict(extractedClaim)` with a `Promise.allSettled` that runs both concurrently:

```typescript
const [verdictSettled, sourcesSettled] = await Promise.allSettled([
  generateEstablishedVerdict(extractedClaim),
  (async () => {
    const allResults = await Promise.all(
      searchQueries.flatMap(query => [
        searchPubMed(query),
        searchCochrane(query),
        searchWhoIris(query),
      ])
    )
    const pubmedBatches = allResults.filter((_, i) => i % 3 === 0)
    const cochraneBatches = allResults.filter((_, i) => i % 3 === 1)
    const whoBatches = allResults.filter((_, i) => i % 3 === 2)
    return deduplicateSources([
      ...cochraneBatches.flat(),
      ...pubmedBatches.flat(),
      ...whoBatches.flat(),
    ])
  })(),
])

if (verdictSettled.status === 'rejected') throw verdictSettled.reason

const sources = sourcesSettled.status === 'fulfilled' ? sourcesSettled.value : []
if (sourcesSettled.status === 'rejected') {
  console.error('Further reading search failed for established claim:', sourcesSettled.reason)
}

return NextResponse.json({ success: true, verdict: { ...verdictSettled.value, sources } })
```

**Key properties:**
- `searchQueries` are already generated by Stage 1 for all claims тАФ no Stage 1 changes needed
- Claude call and search run in parallel: total latency тЙИ `max(~1тАУ2s, ~4тАУ8s)` тЙИ 4тАУ8s (vs current ~1тАУ2s with no sources, and vs ~8тАУ12s for the full research path)
- Search failure is non-fatal: if the search throws, `sources` falls back to `[]` and the established verdict still returns
- If `generateEstablishedVerdict` throws, the existing fallback to the full research pipeline still fires

**`generateEstablishedVerdict` тАФ unchanged.** It still returns `sources: []`; the route merges results in after the fact.

### UI тАФ `VerdictCard` (changed)

Three targeted changes:

**1. Sources section header**

Change from always showing "Sources consulted (N)" to:

```tsx
<p className="text-xs font-mono text-stone-400 uppercase tracking-widest mb-3">
  {verdict.label === 'Established Science'
    ? `Further reading (${verdict.sources.length})`
    : `Sources consulted (${verdict.sources.length})`}
</p>
```

**2. Evidence quality block guard**

Currently hidden when `sources.length === 0`. Now that established claims have sources, switch to a label check:

```tsx
{verdict.label !== 'Established Science' && (
  <div className="animate-fade-slide-up-delay-2">
    <p className="text-xs font-mono text-stone-400 uppercase tracking-widest mb-2">Evidence quality</p>
    <p className="text-stone-600 text-sm">{verdict.evidenceSummary}</p>
  </div>
)}
```

**3. Transparency footer**

Add a third case for established-with-sources. The footer currently branches on `sources.length > 0`. Add an explicit case for `'Established Science'`:

```tsx
{verdict.label === 'Established Science'
  ? 'The explanation above reflects established scientific consensus. The sources below are provided for further reading тАФ they were not used to generate this verdict.'
  : verdict.sources.length > 0
    ? 'This verdict is generated by AI and grounded only in the sources listed above from PubMed/MEDLINE, Cochrane Database, and WHO IRIS.'
    : 'This verdict is generated by AI based on established scientific consensus.'}
{' '}It is not medical advice. Always consult a healthcare professional for personal health decisions.
```

## Data flow

```
Stage 1: extractAndValidateClaim
  тЖТ { isHealthClaim: true, claimType: 'established', extractedClaim, searchQueries }
  тФВ
  тФФтФА claimType === 'established'
       тФВ
       тФЬтФА [parallel] generateEstablishedVerdict(extractedClaim)
       тФВ    тЖТ Verdict { label: 'Established Science', sources: [], context, ... }
       тФВ
       тФФтФА [parallel] 9-fetch fan-out тЖТ dedup
            тЖТ Source[]
       тФВ
       тФФтФА merge: { ...verdict, sources }
            тЖТ Verdict { label: 'Established Science', sources: [...], context, ... }
```

## Error handling

| Failure | Behaviour |
|---------|-----------|
| `generateEstablishedVerdict` throws | Existing fallback: runs full research pipeline |
| Search (any of 9 fetches) throws | Non-fatal: `sources` stays `[]`, established verdict still returned |
| Both throw | Falls back to research pipeline (outer catch) |

---

## What does NOT change

- `generateEstablishedVerdict` function тАФ no changes
- `Verdict` type тАФ no changes
- `VerdictLabel` type тАФ no changes
- Stage 1 (`extractAndValidateClaim`) тАФ no changes
- `searchPubMed`, `searchCochrane`, `searchWhoIris`, `deduplicateSources` тАФ no changes
